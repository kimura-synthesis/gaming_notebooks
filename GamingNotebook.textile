h2. 報告されている諸問題

* nomodeset 問題
* UEFI secure boot

h3. nomodeset 問題

Linux 起動時のカーネルパラメータに nomodeset を追加しないと起動しないという現象

h3. UEFI secure boot

Linux Kernel 4.4.0-20 以降では UEFI secure boot が有効な状態であった場合、
そのポリシーをカーネルが引き継いで、署名無しのドライバがロードされない。

解決方法としては

# UEFI secure boot を無効化
# カーネルで署名無しドライバのロードが出来るように変更

UEFI secure boot 無効化を行うと Windows 10 とのデュアルブート時に Windows 側で
問題が出て来る。

カーネルで署名無しドライバのロードを許可する場合は以下のようにする
<pre>
$ sudo apt install mokutil
$ sudo mokutil --disable-varidation
</pre>

h2. ゲーミングノートブック候補選定

# NVIDIA GTX 10XX 系 GPU 搭載
# DisplayPort 出力有
# ディスプレイのリフレッシュレート
# Ubuntu 動作

Ubuntu 動作に関して問題が出そうなのは NIC が KILLER 製だった場合。
KILLER E2500 は Linux Kernel 4.8 系以降対応なので、Ubuntu 16.04 のデフォルト
カーネル 4.4 系では動かない気がする。
E2400 はカーネル 4.4 系で動くとの話だが、安全を取ると KILLER 製以外の NIC が良いように思える
